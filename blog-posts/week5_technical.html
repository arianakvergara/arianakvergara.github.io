Being CLASSY in Ruby

September 5, 2014

It has been iterated many times but I will say it once again. Everything in Ruby is an object or almost everything. Constructing objects is what we do in Ruby. Classes facilitate this process. A class is a way of organizaing and producing object with similar attributes and methods.

One can create a basic class simply by using the class keyword and the name of the desired class. Conventionally speaking, class names start with a capital letter and use CamelCase instead of any underscores. In my example, I have a class called 'class Jedi'.

A class begins with a method called initialize. The def initialize method is what is being 'booted up' in to each object the class creates. Now we want to name our Jedis so that we can play with a Yoda or Obi-Wan. We do this by creating an instance variable: this instance variable is inherently attached to every instance of the class. Our instance variable for name is @name. Therefore in ruby it is @name=name within the inititalize method. Here, I am creading an instance, yoda, of the class Jedi. yoda has his own @name ("Yoda") and @lightsaber("green"). Those variables belong to the yoda instance which is why they are called instance variables.

Instance variables belong to a particular object and therefore, are better than global variables because they cannot be changed from anywhere from the program. It is a better idea to only be able to change certain variables from a few places.

Class variables are an altogether a different beast. It takes two @@ and therefore is twice as classy. Class variables are created by two @ symbols and are attached to entire classes not just instances of class.
