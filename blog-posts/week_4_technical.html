#cycle

August 31, 2014

I had no idea what Enumerable meant prior to this week. Then I realized, I actually had used Enuemrables before and they are delightful. They make life easier in Ruby and therfore facilitate my own life. Today, I will be discussing the cycle method.

In simple terms, when given a collection, Enumberable #cycle creates an infinitely-repeating ordered source of its items. Ruby expects a block to act on the items and it will infinitely repeat until the break statement is encountered.

#cycle is normally used on an array of some sort. Cycle passes each item to the block and once the last array item has been passed to the block, cycle starts over again at the beginning of the array. Cycle will go through the array forever unless it encounteres a break statement in the block.

You can also use the cycle method on a hash when a block is provided. You can pass each key/value pair to the block as either:

1. A two-element array, with the key element as 0 and its corresponding value as element 1.

2. Two seperate items with the key as the first item and its corresponding value as the second item.

And again as we did earlier with the array we can use the next method to return the next key/value pair in a repeating sequence.
